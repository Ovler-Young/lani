# 系统需求

lani 运行在 64 位 Linux 系统上，在使用基于容器的部署方案时也可以在 Windows 和 MacOS 上使用。M1 Mac 等其他非 amd64 架构的设备暂不支持。

lani 需要互联网连接，其访问的部分服务（如 [TheTVDB](https://thetvdb.com/)）位于中国大陆之外，你可以配置 [代理](#TBA) 来防止请求失败。

### Jellyfin

lani 依赖 [Jellyfin](https://jellyfin.org/)，你需要拥有此服务器的管理员权限。此外还需要进行额外配置，详见 [Jellyfin 配置](#TBA)。

### 下载器

lani 使用 BT 下载动画，目前仅支持 [qBittorrent](https://www.qbittorrent.org/) 作为下载器，未来可能会支持其他下载器。具体配置详见 [qBittorrent 配置](#TBA)。

:::info

由于 lani 需要将动画文件从下载器文件系统复制到 Jellyfin 文件系统，推荐将这三者部署在同一台机器上。否则，你需要使用 [NFS](https://en.wikipedia.org/wiki/Network_File_System) 等网络文件系统将下载器和 Jellyfin 连通。

:::

### 数据库

lani 使用 [PostgreSQL](https://www.postgresql.org/) 作为数据库，最低支持版本为 9.6，推荐使用最新的 LTS 版本。为了使用 lani，你需要 [创建一个 database](https://www.postgresql.org/docs/current/sql-createdatabase.html)，并提供一个拥有该 database 全部权限的用户。

### 对象存储

lani 使用兼容 [AWS S3](https://aws.amazon.com/cn/s3/) 的对象存储来存储图片，你可以在本地部署 [MinIO](https://min.io/)，也可以使用如 [阿里云 OSS](https://www.aliyun.com/product/oss)、[腾讯云 COS](https://cloud.tencent.com/product/cos) 等服务。

别担心！lani 对存储的消耗量很低，几乎不会有任何花费。

在使用 [docker 一体机容器](#TBA) 时，会自动在本地部署 MinIO，无需独立部署。
